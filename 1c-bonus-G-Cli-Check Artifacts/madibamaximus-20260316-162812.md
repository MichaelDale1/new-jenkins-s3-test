## 5. Root Cause Analysis
- Root cause category: Credential drift
- Exact failure mechanism: The application was using incorrect database credentials, leading to access denial.
- Why it wasn’t prevented: The credential drift was not detected by automated monitoring.
- Contributing factors: Manual changes to database credentials without updating the application configuration.

## 6. Resolution
- Actions taken: Updated the database credentials in the application configuration.
- Validation checks: Verified successful database connections and application functionality.
- Evidence of recovery (curl + alarm OK + logs stabilized): Application logs show successful database connections post-fix.

## 7. Preventive Actions
- Immediate (today): Implement automated credential validation checks.
- Short-term (1–2 weeks): Integrate Secrets Manager rotation and validation into CI/CD pipeline.
- Long-term (1–2 months): Conduct a full review of credential management practices and automate drift detection.
